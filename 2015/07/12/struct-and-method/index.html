Struct and Method

12 July 2015 on ruby

It is not the load that breaks you down. It’s the way you carry it. - Lou Holtz

在本篇要介紹的是 Dynamic Classes 以及 Methods，首先先介紹的是 Struct，

Struct

Struct 在 Ruby 裡，就功能上來說是等價於 initialize 的方法，

class Tweet
  attr_accessor :user, :status

  def initialize(user, status)
    @user, @status = user, status
  end
end

#######################################

Tweet = Struct.new(:user, :status)


tweet = Tweet.new('Gregg', 'compiling!')
tweet.user # => Gregg
tweet.status # => compiling!

這兩者就結果上來說是相同的，那如果要在 Struct 裏頭去定義 method 的話，就必須傳入一個 block 進去，

Tweet = Struct.new(:user, :status) do
  def to_s
    "#{user}: #{status}"
  end
end

不過雖然可以這麼做，但是在實作上當然還是建議如果是有涉及到 logic 的部分還是放到 class 裡面，而 Struct 可能就是使用在 attribute 內容的變換。

Alias Method

在開發的過程中，其實經常有機會明明是同樣內容的 method 可是卻不同名稱，這就是 alias_method 的用途，以下面為例，

class Timeline

  def initialize(tweets = [])
    @tweets = tweets
  end

  attr_reader :tweets

  alias_method :contents, :tweets
end

這邊使用了 attr_reader，也就是 create 了一個 getter 的方法，所以 contents 這個方法也就直接複製過來。

Define Method

接下來來談另外一種動態生成的 method 叫做 define_method，

class Tweet
  states = [:draft, :posted, :deleted]

  states.each do |status|
    define_method status do
      @status = status
    end
  end
end

在這個 define_method 下，就可以動態的去生成結構相同但是 attribute 不同的方法。

Send

send 這個方法，是 Ruby 所提供的 syntax sugar 之一，這是任何物件都可以用的方法，然後就可以直接去呼叫該物件的方法，不管他是 public、protected 還是 private，

class Timeline

  def initialize(tweets)
    @tweets = tweets
  end

  def contents
    @tweets
  end

  private

    def direct_messages
    end
end

#######################################
tweets = ['Compiling!', 'Bundling...']
timeline = Timeline.new(tweets)

timeline.contents
timeline.send(:contents)
timeline.send("contents")
timeline.send(:direct_messages)

竟然連 private 的方法都可以直接呼叫，這樣還給不給人隱私啊!!!這樣跟大雄偷看靜香洗澡到底有什麼不同啊!!! 好 der，當然這就是雙面刃，如果要完全無視權限分層，什麼都用 send 呼叫，那其實好像也沒必要，全部用 public 不就好了，所以說畢竟大家都是成年人了(?)還是好好分割權限才比較好維護，不要做什麼偷窺靜香小妹妹的事，應該偷窺 OL

Method Method

最後要提的是 method method，這不是一個中二妹玩疊字的概念，method method 是一個類似 Proc 的方法，也就是說，我們可以把一個物件裡的方法，用 method method 把它存在另外一個物件裡，等到需要的時候在呼叫出來，

class Timeline

  def initialize(tweets)
    @tweets = tweets
  end

  def contents
    @tweets
  end

  def show_tweet(index)
    puts @tweets[index]
  end
end

#######################################
tweets = ['Compiling!', 'Bundling...']
timeline = Timeline.new(tweets)

show_method = timeline.method(:show_tweet)
￼
(0..1).each(&show_method)

show_method.call(0)
show_method.call(1)

基本上在用法上跟 Proc 沒什麼兩樣，甚至如果要把它送進只接收 block 的方法 (像是 each, map)，就一樣是使用 &，將它轉化為 block 送進去，那如果我們在 method method 裡面是放進去 private method 呢？答案依舊是可以被成功呼叫的，Ruby 真是各種後門呢…
