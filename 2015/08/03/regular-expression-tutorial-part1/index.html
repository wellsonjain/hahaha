Regular Expression Tutorial (1)

03 August 2015 on regex, electives

Stop managing your time. Start managing your focus. - Robin Sharma

之前一度對於 Regular Expression 相當頭痛，突然 Code School 就又出了這個新的課程，馬上就開來看，覺得對於初學的人來說真的是相當有幫助，今天就來稍微筆記一下看完整個課程的幾個重要的地方。

The String Story

首先先介紹一下 Regular Expression，又稱作 regex (以下內容都會以 regex 代稱)，是拿來用於檢驗字串是否符合某些規則，regex 在很多程式語言內都被廣泛地運用，舉例來說，像是我們希望一段傳進來的字串符合手機號碼的 pattern 0912345678，但是可能使用者可能習慣輸入 0912-345-678，如果我們希望去檢驗這樣的 pattern 難道我們必須要用 if-else 然後要設計出好幾種狀況，這樣會讓整個 code 寫得很糟糕，所以這時候就是使用 regex 的最佳時機了，

那我們先來看 regex 的格式，

/0911|0912/

由兩個反斜線 / 夾起來，裡面的內容就是我們預期比較的字串是否符合的 pattern，我們這裡定義的意思是他是否有符合 0911 或是 0912，使用一條 Vertical bar | 來表示 OR。

從上面的例子我們可以看得出來，他就是一個字一個字的去比對，但是假設我們有一長串的字串，我們不可能每個字都包含進去，所以我們需要更通用性的方式來套用到多種格式，舉例來說，

/ruby|javascript|html/

如果我們要把各種一個一個加上去好像不太合理，所以接著我們就用更為通用的來實作，

/[a-z]+/

看起來好像有點奇妙，[a-z] 所代表的意思就是一個 a-z 的集合，+ 所代表的意思就是重複他的前一個字元多次，因為我們前面是一個集合，所以就是重複一個集合多次，那就可以組合出任意的字串，但是我們可能有時候也有大小寫的區分，我們可以有兩種方式去寫，一就是在集合裡面再去新增其他的元素，但是還有更方便的方法，就是在最後一個 / 之後面再加上一個我們稱之為 modifier i，i 代表的意思是 insensitive，也就是不分大小寫，不過因為我們也可能會需要空格，像是名字，所以我們在最後一個例子裡面用 \s 代表著 space 的意思，在我們的集合裡面把 space 這個集合給放進去。

/[a-zA-Z]+/
/[a-z]+/i
/[a-z\s]+/i

但是如果像是地址之類的輸入狀況，也有需要輸入數字的可能，那這樣又該怎麼去實作他呢？在這邊我們一樣有兩種做法，一是把數字集合也放進去集合裡面，另一個就是使用通用字符 \w，

/[a-z0-9\s]+/i
/[\w\s]+/

我們看到 \w 甚至連 modifier 都不需要，他代表著 [a-zA-Z0-9] 還有 _。

Crew Emails

接著我們來看一下 email 的格式，這也是相當常見的驗證格式，我們都知道 email 大多就是 hello@world.com 類似這樣的的格式，所以我們可以結合上面所講的，組出一個這樣的 pattern，/\w+@\w+.\w+/，這樣看起來好像蠻合理的，但是這裡面有個很嚴重的問題，就是 . 這個字符在 regex 裡面是屬於通用字符 (除了 \n)，也就是他可以代表任何字符，所以這時候我們就必須用到跳脫字元 \.，

/w+@w+\.(com|net|org|edu)/i

因為可能有很多種 domain 所以我們就把所有可能的 domain 包起來，但是這樣其實有個很容易會讓很 miss 調的問題，就是如果我們的字串是 hello@world.company 這樣也是會符合，因為我們並沒有明確說是否有結尾，所以這邊就要再介紹一下開頭與結尾該怎麼在 regex 表示，^ 代表著開頭，$ 代表結尾所以我們就可以把上面的 regex 給改成這樣，

/^w+@w+\.(com|net|org|edu)$/i
