You might not need jQuery

23 October 2016 on javascript

Launch and learn. Everything is progress. - Danielle LaPorte

jQuery 提供了很多很棒很方便的工具，讓我們在操作 DOM 的時候可以更容易上手，不過隨著新一代的前端框架不停的出現，還有持續更迭的 ECMAScript (JavaScrtipt)，這些在 jQuery 上的 dependency 反而會是一種綁架，也就讓人回頭再看看我能怎麼用 JS 來達成同樣的效果。

最近也是在網路上看到 You might not need jQuery 介紹著一些常用的方法，回到 JS 又該怎麼實作？雖然看了 code 會覺得好像變長不少，但是 jQuery 本身就是抽象化之後的工具，本就是以更簡潔的方式來實作。

在看到這篇文章前先來看看最常見的 jQuery selector mapping 到 JS 之後又會是什麼樣子呢？

#fetches the first matching node only
var q = document.querySelectorAll(".foo");

#fetches a single node by its ID name
var i = document.getElementById("bar");

#fetches nodes matching an element (e.g. h1, p, strong, etc).
var n = document.getElementsByTagName('div') 

#fetches nodes with a specific class name
var c = document.getElementsByClassName('foo')

接著來看最常會使用到的 Ajax，在 jQuery 裡是這樣實作的，

$.getJSON('/my/url', function(data){
  ...
});

而在 JavaScript 可以這麼做，

var request = new XMLHttpRequest();
request.open('GET', '/my/url', true);

request.onload = function() {
  if (request.status >= 200 && request.status < 400) {
    // Success!
    var data = JSON.parse(request.responseText);
  } else {
    // We reached our target server, but it returned an error
  }
};

request.onerror = function() {
  // There was a connection error of some sort
};

request.send();

XMLHttpRequest() 物件 new 出來之後，open() 這個方法顧名思義就是初始化連線的意思，在定義中它包含了幾個參數

void open(
   DOMString method,            # HTTP method
   DOMString url,               # Request url
   optional boolean async,      # true: async, false: sync
   optional DOMString user,     # For authentication
   optional DOMString password  # For authentication
);

在上述的例子是 GET，那如果是 POST 的話就會是下面這樣，不過在例子中的 setRequestHeader() 必須注意必須在 open() 和 send() 之間宣告好，不然就沒帶上了

var request = new XMLHttpRequest();
request.open('POST', '/my/url', true);
request.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded; charset=UTF-8');
request.send(data);

onload() 事件其實蠻好理解，就是在 load 進資料之後要怎麼處理，下面有幾個常見的 event handler 

interface XMLHttpRequestEventTarget : EventTarget {
  // event handlers
  attribute EventHandler onloadstart;
  attribute EventHandler onprogress;
  attribute EventHandler onabort;
  attribute EventHandler onerror;
  attribute EventHandler onload;
  attribute EventHandler ontimeout;
  attribute EventHandler onloadend;
};

最後再以 send() 把請求觸發就好。是說大致的把內部的機制看過真的蠻有趣的，以前可能就比較沒有耐心把它看完，現在對於程式語言的敏銳度比較高一些了，就覺得看著好像就豁然開朗了。
