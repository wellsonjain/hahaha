How to read Source Code

30 May 2016

Everything around you that you call life was made up by people, and you can change it. - Steve Jobs

昨天在臉書上面看到一篇 閱讀程式碼的心理層面 覺得有點感觸，其實自己在閱讀程式碼的方式真的就像是逐條閱讀，誰去呼叫誰？最後預期結果會在哪裡？但是這的確就像是作者所提到「見樹不見林」的狀況；前幾天在韓劇裡面看到一段台詞，大意上是說你所下的每一步棋都有個理由，先有意圖才有行動，回到閱讀原始碼這件事，每個作者對於問題的解法不同，所以應該先去了解他對於整個程式的架構組件是如何去設計的，要把持住自己對於細節的慾望，目標導向「我的意圖是什麼？」，「我想要瞭解的地方是哪裡？」

看完了這篇文章之後，心裡的好奇心就被激發出來了，於是乎就去找了一下是否有解釋如何閱讀程式碼之類的文章，結果找到了這篇 iThome 在 2008 年初由作者王建興所撰寫的 如何閱讀程式碼的系列文 在這個系列文中，作者也是提出了先了解架構，採由上至下 (Top-Down) 的方式，從骨幹開始了解，去釐清開發者思考的脈絡，他選擇如何鋪陳。

雖然後續也找了很多篇來看，不過我想核心價值應該都是一樣的，過去自己有個很大的問題是，「總是希望有步驟可以遵循」，其實很少從 Top-Down 的方式來去審視整個結構，就像是身處於一個陌生的城市，若是要從 A 點到 B 點，沿途一直問人也是很耗腦力，可是如果從地圖上來看自己的位置然後又要到哪裡去，雖然這樣的類比看似很容易，但是在閱讀程式碼的過程還是必須刻意去練習才能做到，畢竟 Source Code 也是人類心智下的產物。

總結一下，理解開發者的設計架構，想清楚自己的目的意圖是什麼？而 Source Code 是如何去達成這個目的的？
